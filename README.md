## Контрольні питання 

**1. Призначення та архітектура JFC**: JFC — це комплексний фреймворк для розроблення графічних інтерфейсів у Java. Його архітектура включає AWT, Swing, Java 2D та Accessibility API.

**2. Переваги Swing порівняно з AWT**: Компоненти Swing є легковаговими (lightweight), кросплатформними та мають ширший набір елементів. AWT натомість використовує важкі компоненти, що залежать від конкретної операційної системи.

**3. Структура GUI програми**: Типова структура складається з головного вікна (`JFrame`), проміжних контейнерів для групування елементів (`JPanel`) та базових компонентів для взаємодії з користувачем (`JButton`, `JTextField`).

**4. Призначення Containers**: Контейнери використовуються для розміщення та організації інших компонентів за допомогою менеджерів компонування (Layout Managers). В якості контейнерів виступають `JFrame`, `JPanel`, `JDialog`, `JWindow`.

**5. Основні методи класу Frame**: Найчастіше використовуються `setSize(width, height)` для встановлення розміру, `setVisible(boolean)` для відображення на екрані, `setTitle(String)` для назви вікна та `setLayout(LayoutManager)` для вибору менеджера компонування.

**6. Призначення об'єкта Panel**: `Panel` (або `JPanel` у Swing) слугує для логічного групування компонентів в окремі блоки. Елементи додаються до панелі за допомогою методу `add(Component)`.

**7. Об'єкти типу event та механізм оброблення подій**: Об'єкти подій фіксують дії користувача (клік мишкою, натискання клавіші). Механізм працює за моделлю делегування: компонент генерує подію та передає її зареєстрованим слухачам (Listeners), які виконують відповідний код.

**8. Призначення інтерфейсів XxxListener**: Ці інтерфейси містять методи для обробки конкретних подій. Типові приклади: `ActionListener` (для натискання кнопок), `MouseListener` (для дій миші), `KeyListener` (для клавіатури).
